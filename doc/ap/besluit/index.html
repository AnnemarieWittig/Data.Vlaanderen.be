Dit document beschrijft een applicatieprofiel voor Besluit, in dit geval Besluit Publicatie AP. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Status van dit document

Deze sectie beschrijft de status van dit document op het moment van publicatie. Het kan zijn dat er ondertussen al nieuwere versies zijn uitgebracht.

Dit applicatieprofiel dient om het specifieke gebruik van de entitieten relevant voor de beschreven applicatie te verduidelijken.

Agendapunt

Formeel aanvaard te behandelen onderwerp op een zitting.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: titel, beschrijving, geplandOpenbaar, type, refereert aan, aangebracht na, heeft ontwerp besluit.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
titel String 1 ... 1 De naam van het punt.
beschrijving String 0 ... 1 Korte beschrijving van het punt.
gepland openbaar Boolean 1 ... 1 Of het agendapunt al dan niet bedoeld is om openbaar te bespreken.
type Code 1 ... 1 Type van het agendapunt.
refereert aan Agendapunt 0 ... * Een ander agendapunt waarnaar verwezen wordt dit punt.
aangebracht na Agendapunt 0 ... 1 Het agendapunt dat op de agenda direct dit punt voorafging.
heeft ontwerp besluit Document 0 ... * Een (mogelijks nog aan te vullen) ontwerp voor het besluit dat uit dit agendapunt zou voortkomen.

Artikel

Formeel afgebakend onderdeel van een besluit, dat een of meer van de beoogde rechtsgevolgen beschrijft.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: nummer, realiseert.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
nummer String 1 ... 1 Het nummer van het artikel.
realiseert Legale Resource (Artikel) 1 ... 1 Linkt een legale verschijningsvorm naar de legale resource gerealizeerd in die verschijningsvorm.

Behandeling van Agendapunt

De uitvoering van de acties om een agendapunt te verwerken tijdens een zitting.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: openbaar, heeft voorzitter, heeft secretaris, gebeurt na, heeft aanleiding tot, heeft aanwezige, heeft onderwerp, heeft stemming.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
openbaar Boolean 1 ... 1 Of de bespreking effectief openbaar verlopen is.
heeft voorzitter Mandataris 0 ... 1 De mandataris die de rol als voorzitter heeft gedurende de.behandeling van het punt. Enkel te gebruiken wanneer de voorzitter tijdens dit punt verschilt van de voorzitter van de zitting.
heeft secretaris Mandataris 0 ... 1 De mandataris die de rol als secretaris heeft gedurende de behandeling van het punt. Enkel te gebruiken wanneer de secretaris tijdens dit punt verschilt van de secretaris van de zitting.
gebeurt na Behandeling van Agendapunt 0 ... 1 Verwijzing naar de voorgaande behandeling binnen dezelfde zitting. Laat toe om de volgorde van de behandelingen op te bouwen.
heeft aanleiding tot Besluit 0 ... * Een besluit dat is opgemaakt naar aanleiding van de behandeling van het agendapunt.
heeft aanwezige Mandataris 1 ... * Een mandataris (van het orgaan dat de zitting houdt) die aanwezig was tijdens een deel van de behandeling.
onderwerp Agendapunt 1 ... 1 Het onderwerp van de activiteit.
heeft stemming Stemming 0 ... * Een stemming die plaatsvond tijdens de behandeling van het agendapunt.

Besluit

De authentieke schriftelijke neerslag van een beslissing van een bestuursorgaan. Deze beslissing houdt een rechtshandeling in waarbij sprake is van een beoogd rechtsgevolg.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: samenvatting, citeeropschrift, motivering, publicatiedatum, realiseert, volgt uit.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
beschrijving String 0 ... 1 Een beknopte beschrijving van het besluit.
citeeropschrift String 0 ... 1 Een korte benaming die in praktijk gebruikt wordt om naar de resource te verwijzen.
motivering LangString 1 ... 1 Beschrijving van de motivering achter de beslissing die wordt uitgedrukt in het besluit.
publicatiedatum Date 1 ... 1 De officiële publicatiedatum van de regelgeving.
realiseert Legale Resource (Besluit) 1 ... 1 Linkt een legale verschijningsvorm naar de legale resource gerealizeerd in die verschijningsvorm.
volgt uit Behandeling van Agendapunt 1 ... 1 De agendapunt behandeling die tot dit besluit heeft geleid.

Bestuursorgaan

Een formeel orgaan van een rechtspersoon dat bevoegd is om rechtshandelingen te stellen.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: naam.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
naam Literal 1 ... 1 Naam van de eenheid.

Legale Resource

Een legaal concept dat los staat van een bepaalde vorm (het achterliggende idee). Een legale resource kan zowel een geheel (bvb: wet, besluit) voorstellen als een onderdeel eruit (bvb: artikel).

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: type document, gecorrigeerd door, citeert, heeft bijlage, veranderd door, verandert, corrigeert.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
type document ResourceType 1 ... * Het type van de legale resource. (Voorbeelden: "reglement", "wet", "voorstel"...)
gecorrigeerd door Legale Resource 0 ... * Duidt een resource aan die een tekstuele verandering zonder legale impact aan deze resource aanbrengt.
citeert Legale Resource 0 ... * Een citatie in de wettelijke tekst. Dit omvat zowel woordelijke citaten als citaten in verwijzingen.
heeft bijlage Document 0 ... * Een bijlage waarnaar verwezen wordt.
veranderd door Legale Resource 0 ... * Duidt aan dat deze resource een verandering met legale gevolgen is ondergaan door een andere resource. Dit omvat amendementen, vervangingen, herroepingen en andere types verandering.
verandert Legale Resource 0 ... * Duidt aan dat deze resource een verandering met legale gevolgen doorvoert op een andere resource. Dit omvat amendementen, vervangingen, herroepingen en andere types verandering.
corrigeert Legale Resource 0 ... * Duidt aan dat deze resource tekstuele veranderingen zonder legale impact aanbrengt in een andere resource.

Legale Resource (Artikel)

Het legale concept van een artikel dat los staat van een bepaalde vorm (het achterliggende idee).

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: is deel van.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
is deel van Legale Resource (Besluit) 1 ... 1 De resource waar deze resource (als fysiek geheel) onderdeel van uitmaakt.

Legale Resource (Besluit)

Het legale concept van een besluit dat los staat van een bepaalde vorm (het achterliggende idee).

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: aangenomen door, heeft deel.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aangenomen door Bestuursorgaan 1 ... 1 Het orgaan dat de legale resource heeft goedgekeurd, waardoor er een wettelijk gevolg onstond.
heeft deel Legale Resource (Artikel) 0 ... * Duidt een resource aan die een (fysiek) onderdeel voorstelt van deze resource.

Legale Verschijningsvorm

Een legale verschijningsvorm is een intellectuele realisatie van een legale resource onder de vorm van een opeenvolging van tekens (meestal alphanumerieke tekens).

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: titel, inhoud, inwerkingtreding, opheffing, taal.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
titel String 1 ... * Titel van de resource. Voor een besluit is dit het opschrift, voor een artikel de benaming van het artikel.
inhoud Literal 1 ... 1 De legale inhoud.
inwerkingtreding Date 1 ... 1 De datum waarop de regelgeving van kracht wordt. Dit is niet hetzelfde als wanneer de regelgeving in werking treedt. De toepasbaarheid van de regelgeving kan op of na de inwerkingtreding liggen.
opheffing Date 0 ... 1 De laatste dag waarop de regelgeving nog van kracht is.
taal Taal 1 ... 1 De taal van de verschijningsvorm. Deze property is aanwezig voor compatibiliteit met European Legislation Identifier (ELI). Link

Mandataris

Een persoon in zijn rol als houder van een mandaat.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: start, einde, beleidsdomein, status, rangorde.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
start DateTime 1 ... 1 Tijdstip wanneer de mandataris het mandaat opneemt.
einde DateTime 1 ... 1 Tijdstip wanneer de mandataris het mandaat niet langer opneemt.
beleidsdomein Code 0 ... * Het thema of beleidsdomein dat toegewezen is aan een mandataris.
status Code 1 ... 1 De gesteldheid van de mandataris.
rangorde LangString 0 ... 1 De aanduiding van de positie binnen de machtsorde van het bestuursorgaan waarin de mandataris zetelt.

Stemming

Geformaliseerde handeling waarbij de stemgerechtigde leden van een bestuursorgaan hun keuze te kennen geven.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: geheim, resultaat, onderwerp, heeft voorstander, heeft aanwezige, heeft tegenstander, heeft stemmer, heeft onthouder.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
geheim Boolean 1 ... 1 Of de stemming geheim was. Bij een geheime stemming wordt niet individueel bekend gemaakt wie op wat heeft gestemd.
resultaat LangString 1 ... 1 De uitslag van de stemming.
onderwerp LangString 1 ... 1 De beschrijving van het onderwerp waarover de stemming gaat.
heeft voorstander Mandataris 0 ... * Mandataris die als voorstander heeft gestemd op het onderwerp van de stemming.
heeft aanwezige Mandataris 1 ... * Een mandataris (van het orgaan dat de zitting houdt) die aanwezig was tijdens de stemming.
heeft tegenstander Mandataris 0 ... * Mandataris die tegen het onderwerp van de stemming heeft gestemd.
heeft stemmer Mandataris 1 ... * Een mandataris die deelneemt aan de stemming. In geval van een niet-geheime stemming kunnen de meer gespecialisseerde versies (bvb. heeft voorstander) gebruikt worden.
heeft onthouder Mandataris 0 ... * Mandataris die heeft deelgenomen aan de stemming met een onthouding. Dit omvat ook ongeldige stemmen, blanco stemmen of het weigeren deel te nemen aan de stemming.

Vergaderactiviteit

Een vergadering of fase ervan.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: heeft aanwezige.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
heeft aanwezige Agent 0 ... * Een agent die aanwezig was tijdens een deel van de activiteit waar hij een zekere verantwoordelijkheid had.

Zitting

Een geformaliseerde samenkomst van de leden van een bestuurorgaan met het doel om de aangelegenheden te regelen waarvoor het bevoegd is.

Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: start, eind, locatie, heeft notulen, heeft secretaris, heeft zittingsverslag, heeft voorzitter, behandelt, heeft aanwezige bij start, is gehouden door.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
gestart op tijdstip DateTime 1 ... 1 Tijdstip waarop de zitting begint.
geëindigd op tijdstip DateTime 1 ... 1 Tijdstip waarop de zitting eindigt.
op locatie Locatie 0 ... 1 Locatie waar de zitting plaats vindt. Doorgaans is dit een zaal in een gebouw.
heeft notulen Document 0 ... 1 De formele, volledige notulen van de zitting.
heeft secretaris Mandataris 1 ... 1 De mandataris die de rol als secretaris heeft gedurende de zitting.
heeft zittingsverslag Document 0 ... * Een verslag van de zitting.
heeft voorzitter Mandataris 1 ... 1 De mandataris die de rol als voorzitter heeft gedurende de zitting.
behandelt Agendapunt 0 ... * Een formeel vastgelegd punt van de zitting.
heeft aanwezige bij start Mandataris 1 ... * Een mandataris (van het bestuurorgaan die de zitting hield) die aanwezig was bij de start van de zitting.
is gehouden door Bestuursorgaan 1 ... 1 Duidt het orgaan aan van welke de zitting is gehouden.
